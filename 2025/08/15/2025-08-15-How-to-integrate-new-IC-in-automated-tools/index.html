How to add ICx in Integration Builder

Assume that IC name is ICx.

Compilation logic

Integration Process (by module)

1.Add ICx info in SUPPORTED_IC_MAP under common folder, which contains many base classes and common functions.

2.Create builder_ICx.py under build folder: It is the implementation of the entire compilation logic.
Each IC has similar logic implementation, thereby the py files are also similar.

3.Create xml file for ICx, which contains all the testcase info, like tc_name.

4.Create tc_build_config_base_ICx.py under config/config_base: Configure for image building.

5.Create jenkins_test_target_config_ICx.py under ic_configs: Some configurations bound to the SDK.
Retrieve and assemble the compilation and packaging information of the application from a specific configuration. The returned result is a dictionary, which contains the key information needed for subsequent automated builds, scripts, pipelines, or debugging.

How to add ICx in Integration Tester

Assume that IC name is ICx.

Test logic

Integration Process (by module)

1.Add common info if needed.

2.Modify .py under config folder
Add class ICx, which contains the configure info for ICx

3.Add ic_configs/xx/ICx/ic_default_config.py
Update the info for ICx under ic_configs

4.Create xml file for ICx, which contains all the test cases
builder tool: Search for config by xml, for building;
tester tool: search for test process by xml.

5.Modify soc_device.py under test_utils/devices/ if needed

6.Modyify tester_helper_base.py under test_utils/test_helper/

7.Create tester_helper_ICx.py under test_utils/test_helper/
It’s the main py module to achieve key functions like download, reset, etc.

8.Create auxiliary/.py
Encapsulate all input parameters as class

9.Modify get_device_setting.py under test_utils/test_common_lib/
Parse parameters，and pass to above class.
