Part1：PDM->SAI

1、Build Yocto

Please set your own path in advance.

Download Yocto

$repo init -u https://source.codeaurora.org/external/imx/imx-manifest -b imx-linux-zeus -m imx-5.4.24-2.1.0.xml

$repo sync

$ls

$cp 0001-meta-bsp-add-support-to-run-lk-with-jailhouse.patch sources/meta-imx/

$cd source/meta-imx/

Apply patch

$git am 0001-meta-bsp-add-support-to-run-lk-with-jailhouse.patch

Compile Yocto

$bitbake fsl-image-validation-imx

$DISTRO=fsl-imx-xwayland MACHINE= source imx-setup-release.sh -b build

Note: MACHINE=

Replace the machine name,which is depended on the development board you are using.

2、Build linux

Please set your own path in advance.

$git clone ssh://git@bitbucket.sw.nxp.com/~nxa26470/linux-imx.git

//change path

$cd harpoon/

$mkdir kernel

$cd kernel/

Download linux

$git clone git://source.codeaurora.org/external/imx/linux-imx -b imx_5.4.24_2.1.0

*Apply patch *

put 0001-dts-add-imx8mn-ddr4-evk-lk.dts-to-support-jailhouse-.patch under linux-imx/

$git am 0001-dts-add-imx8mn-ddr4-evk-lk.dts-to-support-jailhouse-.patch

Compile linux

$cd linux-imx/

$source /opt/fsl-imx-xwayland/5.4-zeus/environment-setup-aarch64-poky-linux

$unset LDFLAGS

$make defconfig

$make -j8

3、Build imx-jailhouse

With the Jailhouse hypervisor, two operating systems (Linux and Little Kernel) can run simultaneously on the IMX8MN chip. Jailhouse is a lite virtual machine that can load/run LK and allocate hardware resources to LK and Linux.

In this partition, LK handles the tasks related to audio capture, mixer, and playback, and Linux handles the tasks related to EMMC /USB/HMI.

Download imx-jailhouse

Please set your own path in advance.

$git clone https://source.codeaurora.org/external/imx/imx-jailhouse -b imx_5.4.24_2.1.0

*Apply patch *

Put bellowing patch under

- 0001-arm-irqchip-fix-irq_bitmap-setting.patch

- 0002-configs-add-imx8mn-lk.c-for-little-kernel-cell.patch

- 0003-configs-change-imx8mn-lk.c-and-imx8mn.c-about-ivshme.patch

- 0004-scripts-add-jstart.sh-and-jkill.sh-to-run-or-stop-li.patch

$cd imx-jailhouse/

$git am 0001-arm-irqchip-fix-irq_bitmap-setting.patch

$git am 0002-configs-add-imx8mn-lk.c-for-little-kernel-cell.patch

$git am 0003-configs-change-imx8mn-lk.c-and-imx8mn.c-about-ivshme.patch

$git am 0004-scripts-add-jstart.sh-and-jkill.sh-to-run-or-stop-li.patch

Compile imx-jailhouse

$ cd harpoon/

$ cd imx-jailhouse/

$ source /opt/fsl-imx-xwayland/5.4-zeus/environment-setup-aarch64-poky-linux

$ unset LDFLAGS

$ export KDIR=

$ make CC=”$CC”

Note：

$export KDIR=

Replace with the appropriate path.

4、Build LK

Please set your own path in advance.

Install toolchain of lk

$wget https://releases.linaro.org/components/toolchain/binaries/7.3-2018.05/aarch64-elf/gcc-linaro7.3.1-2018.05-x86_64_aarch64-elf.tar.xz

$tar xf gcc-linaro-7.3.1-2018.05-x86_64_aarch64-elf.tar.xz

please make sure gcc-linaro-7.3.1-2018.05-x86_64_aarch64-elf is installed under /opt/ when you compile lk and meet can’t find correct toolchain problem, please check file imx8/lk_inc.mk.

Compile little kernel

$cd Harpoon-R1.0/Harpoon/imx8/

$make

Note: The default code is designed for pdm-input case, for sai-input case, you need to modify below codes before ‘make’ command:

1) change ak5558 {…status = “disabled”;} to ak5558 {…status = “ok”;} in target/imx8mn/evk/imx8mnevk.dts

2) change &sai5 { status=” disabled”; …} to &sai5 {status=”ok”; …} in target/imx8mn/evk/imx8mnevk.dts

3) change #define PDM_INPUT to //#define PDM_INPUT in app/audio_mixer/audio_mixer.c

5、Install binary

Using Win32 software copies the.wic file to the SDK, then transfers it to the board and runs it.

Check the path in output part of each step above to find the corresponding generated files, copy them to the USB disk (excluding WIC file), and then send it to the corresponding development board.

The board is divided into two parts, the SDK part and the boot part, copy all files from USB flash drive to SDK, and the Image and lk.dtb files are copied from the SDK to the boot.

Now boot the board and hit enter during the u-boot countdown. This is only needed during the first boot.

On the u-boot prompt enter:

u-boot=> setenv fdt_file imx8mn-ddr4-evk-lk.dtb

u-boot=> saveenv

u-boot=> boot

Then you can hook up the microphone，and do test：

Execute command - linux console : ./jstart.sh

Execute command - lk console ：mixer

Part2：SAI->SAI

1、Differences

For SAI->SAI, you need to change the codes and the development board（this requires the Audio Board）, the differences between part1 and part2 are as follows:

\1. One is that the code is different, and there are three differences that need to be changed manually.

\2. The other thing is that you should use an audio version instead of a microphone.

\3. There is no need to edit Yocto, because the place where the code is changed is LK instead of Yocto, so LK needs to be recompiled.

2、Code change steps

*Compile little kernel *：

$ cd harpoon/

$ cd imx8/

$make

Note: The default code is designed for pdm-input case, for sai-input case, you need to modify below codes before ‘make’ command:

1) change ak5558 {…status = “disabled”;} to ak5558 {…status = “ok”;} in target/imx8mn/evk/imx8mnevk.dts

2) change &sai5 { status=” disabled”; …} to &sai5 {status=”ok”; …} in target/imx8mn/evk/imx8mnevk.dts

3) change #define PDM_INPUT to //#define PDM_INPUT in app/audio_mixer/audio_mixer.c

Please use the Vim command to modify the corresponding code.
