Compilation steps

Kernel

$ git clone ssh://…

//Clone the codes from corresponding link you have, enter the command: git clone “SSH +site”, the code will be cloned successfully

//Pay attention to the path

$ cd linux-imx

$ export PATH=${PATH}:/opt/tizen5.5_cross_toolchain_vd_kant_target_armv7l_host_x86-64/bin/

$ make ARCH=arm CROSS_COMPILE=armv7l-tizen-linux-gnueabi- iMX8M-Tizen_debug_defconfig

$make ARCH=arm CROSS_COMPILE=armv7l-tizen-linux-gnueabi- uImage LOADADDR=0x40008000 -j32

$ make ARCH=arm CROSS_COMPILE=armv7l-tizen-linux-gnueabi- -j32

Jailhouse

//Pay attention to the path

$ cd imx-jailhouse

$ export CC=”armv7l-tizen-linux-gnueabi-gcc -static”

$ export PATH=${PATH}:/opt/tizen5.5_cross_toolchain_vd_kant_target_armv7l_host_x86-64/bin/

Jailhouse ARM32 binary and kernel module

$make KDIR=../linux-imx/ CC=”$CC” ARCH=arm CROSS_COMPILE=armv7l-tizen-linux-gnueabi- clean

$ make KDIR=../linux-imx/ CC=”$CC” ARCH=arm CROSS_COMPILE=armv7l-tizen-linux-gnueabi-

Jailhouse ARM64 binaries and imx8mm config files

$ source /opt/fsl-imx-xwayland/5.4-zeus/environment-setup-aarch64-poky-linux

$ unset LDFLAGS

$ unset PKG_CONFIG_PATH

$ make ARCH=arm64 CROSS_COMPILE=aarch64-poky-linux- KDIR=../linux-imx/ CC=”aarch64-poky-linux-gcc –sysroot=/opt/fsl-imx-xwayland/5.4-zeus/sysroots/aarch64-poky-linux” clean

$ make ARCH=arm64 CROSS_COMPILE=aarch64-poky-linux- KDIR=../linux-imx/ CC=”aarch64-poky-linux-gcc –sysroot=/opt/fsl-imx-xwayland/5.4-zeus/sysroots/aarch64-poky-linux”

Jailhouse-services

//Pay attention to the path

$ cd jailhouse-services

$ export CC=”armv7l-tizen-linux-gnueabi-gcc -static”

$ export PATH=${PATH}:/opt/tizen5.5_cross_toolchain_vd_kant_target_armv7l_host_x86-64/bin/

$ make KDIR=../linux-imx/ CC=”$CC” LKDIR=../imx8/ IVSHM_V2=y ARCH=arm CROSS_COMPILE=armv7l-tizen-linux-gnueabi- AFDIR=../af clean

$ make KDIR=../linux-imx/ CC=”$CC” LKDIR=../imx8/ IVSHM_V2=y ARCH=arm CROSS_COMPILE=armv7l-tizen-linux-gnueabi- AFDIR=../af modules

AF

$ cd af

$ make imx8mm_sbp_defconfig

$ make menuconfig

//Enable the PDM SW Decimation

$ unset CFLAGS

$ scripts/build_virt.sh imx8mm

Generated Image

According to the different path defined by each developer, for each compilation step，the corresponding files in the board need to be replaced by the files generated after each compilation step.

Notice

1.Pay attention to the path for each step.

2.Check whether the generated file is generated by yourself: “ls -l +filename”.

3.Error analysis:Think about the steps made up by others and think about the differences would be. It is likely that you have different paths.

4.Using “ls” command more often，check what’s in the current folder at any time.

5.If you need to modify the code, enter the relevant file through the “vim” command, save and exit after modification, and then recompile.

6.The “.o file” refers to the generated file.
