Release Test

(1)Test purpose

Check for problems with the new SDK release.

(2)Pre-test preparation

Download files that needed.

Then copy the files to a USB.

(3)Test items and steps

1. Check if the board can boot normally

You can check if there is any error/crash log in Linux and LK (little kernel) side：

Open device manager and check the serial port.

Open the corresponding two serial ports, one is for Linux and the other is for little kernel.

2. Environment configuration

1>Flash the file into the board

Note：If you’re updating file to the board for the first time, you need to update it first.

2> Update the new release package using cp command

3. Reboot the board

4. Check version id in lk log

After board reboot and lk start successfully, type “version” command in lk side, compare the git revisions & software version id to the id number in downloaded file.

5. Remember turning on the microphone before each test

6. BT

Use remote control to change source to “Bluetooth”, then connect soundbar with your mobile phone or TV or other device which has BT function, then play some music to check if it can connect, play normally, no noise and no error log.

7. ACM/WIFI

Use remote control to change source to “WIFI”, then connect soundbar and TV with the same wifi, then play some music to check if it can connect, play normally, no noise and no error log.

8. HDMI

Connect soundbar “hdmi-in” port to pc or Blu-ray, use remote-control to change source to “HDMI”, if you use pc, you can use vlc player to check if different format audio can play normally and transition has no error.

9. Spdif

Connect soundbar “spdif” port to pc or Blu-ray, use remote-control to change source to spdif, if you use pc, use vlc player to check if different format(PCM\DD\DTS) audio can play normally.

10. ALSA(alsa-input)

1> Switch source to alsa-input mode firstly

2>Copy a PCM audio to a USB drive, and then flash it into the soundbar

3>Play the audio stream with aplay

11. Afvoice

Make sure you’ve already copied XXXX.wav to the board, then input the aplay or paplay command.

12. Alexa

1> You can follow below guide to test Alexa , as for amazon account, you can register a new one.You need to input username and password correctly; and please use correct wifi ap: name: ** ,password: **.

2> And then, go to Amazon.com in your browser and enter your Amazon account and password. Besides, you also need a password obtained from another two command.

3> The last, if all of the steps are correct, you can check the Alexa function now.

You can ask some questions to Alexa, such as : What’s the weather?

13. Event manager
