Release Test

(1) Test purpose

Check for problems with the new SDK release.

(2) Pre-test preparation

  • Download needed files.
  • Copy the files to a USB drive.

(3) Test items and steps

1. Check if the board can boot normally

  • Check for any error/crash log in Linux and LK (little kernel) side.
  • Open device manager and check the serial port.
  • Open the corresponding two serial ports: one for Linux, one for little kernel.

2. Environment configuration

1> Flash the file into the board

Note: If you’re updating file to the board for the first time, you need to update it first.

2> Update the new release package using cp command

3. Reboot the board

4. Check version id in lk log

  • After board reboot and LK has started successfully, type the version command on LK side.
  • Compare the git revision & software version id to the downloaded file’s id.

5. Remember turning on the microphone before each test

6. BT (Bluetooth)

  • Use remote control to switch source to “Bluetooth”.
  • Connect soundbar with your mobile phone/TV/other BT device.
  • Play music to make sure it connects and plays normally, without noise or error log.

7. ACM/WIFI

  • Use remote control to switch source to “WIFI”.
  • Connect the soundbar and TV to the same wifi network.
  • Play music to check if it connects/plays normally, no noise or error log.

8. HDMI

  • Connect soundbar “hdmi-in” port to PC or Blu-ray.
  • Use remote control to switch source to “HDMI”.
  • For PC, use VLC player to test playback of different audio formats and check for normal operation and error-free transitions.

9. Spdif

  • Connect soundbar “spdif” port to PC or Blu-ray.
  • Remote control “Spdif” source.
  • For PC, check with VLC player for different formats (PCM/DD/DTS) to ensure normal playback.

10. ALSA (alsa-input)

1> Switch source to alsa-input mode first

2> Copy a PCM audio file to a USB drive, then flash it into the soundbar

3> Play the audio stream with aplay

11. Afvoice

  • Make sure you’ve copied the needed XXXX.wav file to the board.
  • Use aplay or paplay command to play.

12. Alexa

  1. Follow the guide to test Alexa. You can register a new Amazon account if needed. Input username and password correctly; connect to correct wifi AP.
  2. Go to Amazon.com in your browser, log in with your Amazon account. You may need a password from other two commands.
  3. If all steps are finished, you can check Alexa functionality. For example, ask: “What’s the weather?”

13. Event manager

  • Set the pipeline, and read back the pipe settings with the kcontrol command.

1> Switch source to alsa-input mode (other mode also works)

2> Set the pipeline

3> Read back the pipe settings
