上行功率回退

全称：Upstream power back-off（简称 UPBO）
作用：执行 UPBO，改善（不同长度回路）系统间的频谱性能。（握手阶段不使用UPBO这一机制）

降低 VTU-R 的发送 PSD 原理

  1. VTU-R 的发送 PSD 掩模（PSDMASKus）由 VTU-O 计算。
  2. 初始化时（O-SIGNATURE 消息中），将 PSDMASKus 传递给 VTU-R。
  3. VTU-R 按 UPBO 原理，自行完成 UPBO，无需向 VTU-O 发送任何有用信息。
  4. 加 UPBO 后，VTU-O 必须能调整 VTU-R 所选择的发送 PSD。

PSD 掩模（UPBOMASK）

  • 初始化时，VTU-R 必须明确估计环路电长度（klo），由 klo 计算 UPBOMASK。
  • 初始化和数据传送时期，VTU-R 必须使其发送信号自适应 PSDMASK（klo）的要求。
  • 其余时段，根据 PSDMASKus 限值和上行PSD上限，计算 UPBOMASK：

UPBOMASK(klo, f) = UPBOPSD(f) + LOSS(klo, f) + 3.5 [dBm/Hz]

其中，

  • LOSS(klo, f) = klo * sqrt(f) [dB]
  • UPBOPSD(f) = -a - b * sqrt(f) [dBm/Hz]

UPBOPSD(f) 是频率的函数，与环路长度和类型无关，f 用 MHz 表示。每个上行频带的 a 和 b 不同，由 CO-MIB 给定，并在初始化期间提供给 VTU-R。

klo 估算算法

若估算出的 klo < 1.8，则允许调制解调器按照 klo = 1.8 执行回退。klo 估算主要方法如下：

(1) M0 (default method)

直接计算 1MHz 附近的 klo：

  • 由 klo = min(LOSS(f) / sqrt(f)) 计算 Actual KL0-CPE value (0.1dB) 和 Actual KLO-CO value (0.1dB)。

其中，Actual KL0-CPE value(0.1dB) 由cpe算好后传递过来。

(2) M1 (alternative method)

不同 Mode 有不同算法：

- Mode 0

M0 方法计算的 1MHz 的 klo 赋给每个 band。

  • klo = MIN(LOSS(f) / sqrt(f))

- Mode 1

CPE 完成：

  • ELE[band] = PERCENTILE({loss(f, rx_thresh(band)) / sqrt(f)}, UPBOELMT) [dB]
  • ELEDS = MIN(ELE[band])（band 属于下行 band）
  • klo[band] = ELEDS [dB]

即 CPE 算出 DS1、DS2、DS3 的 klo，取最小赋给所有上行 band。

- Mode 2

CO 完成：

  • ELE[band] = PERCENTILE({loss(f, rx_thresh(band)) / sqrt(f)}, UPBOELMT) [dB]
  • klo[band] = ELE[band] [dB]（CO 算出 US2、US3 两个 band 的 klo，分别赋值）

- Mode 3

  • klo[band] = ELE[band] [dB]
  • ELEUS = MIN(ELE[band])（上行 band）
  • ELEDS = MIN(ELE[band])（下行 band）
  • 取 ELEUS 和 ELEDS 的最小值赋给上行每个 band。

其他说明与注意

  • klo 估计值是 min[loss(f) / sqrt(f)]，在 1MHz 以上可取最小值。
  • loss（损耗）是该环路在频率 f 上的插入损耗（dB）。
  • 多根电缆捆扎情况下，随着环路缩短，FEXT 快速减小；klo < 1.8 时，不建议再回退。
  • 电长度 1.8 相当于 70m 的 0.4mm 环路。

常用术语

  • CO：局端
  • CPE：终端
  • VTU-O / R：系统收发信机单元（远端/近端）
